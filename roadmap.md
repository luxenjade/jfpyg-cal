# 🗺️ JFPYG cal 開発ロードマップ (Netlify前提)

DBを持たない、Googleカレンダー印刷特化型Webアプリ「JFPYG cal」の構築ロードマップ。
React + shadcn/ui + Vite + TypeScript で実装し、NetlifyのAIビルド分析を味方につけて爆速で開発を完了させます。

---

## 📅 フェーズ 1: 環境構築とベースUIの配置

まずは開発基盤を整え、shadcn/ui を使ってアプリの見た目の骨組みを作ります。

- [ ] **1-1. Viteプロジェクトの起ち上げ**
  - TypeScript環境でViteプロジェクトを作成する。
  - 不要な初期ファイル（`App.css` などのテンプレート）をクリーンアップ。
- [ ] **1-2. Tailwind CSS & shadcn/ui の導入**
  - Tailwind CSSをセットアップ。
  - `npx shadcn@latest init` で shadcn/ui を初期化。
  - 必要となるコンポーネント（`Button`, `Checkbox`, `ScrollArea`, `Dialog` 等）をインストール。
- [ ] **1-3. 基本レイアウトの作成**
  - 画面を「左側：カレンダー選択＆操作サイドバー」「右側：A4印刷プレビュー画面」の2カラムに分割（レスポンシブ考慮）。
  - Netlifyのリロード対策として `public/_redirects` ファイルを作成し、`/* /index.html 200` を記述。

---

## 🔐 フェーズ 2: Google API 連携 & 認証（DBなしの核）

ユーザーのブラウザと Google API を直接繋ぎ、セッションを保持しない安全なデータ取得フローを作ります。

- [ ] **2-1. Google Cloud Console の設定**
  - Google Cloudで「JFPYG cal」プロジェクトを作成。
  - **Google Calendar API** を有効化。
  - OAuth 同意画面を設定（スコープは `.../auth/calendar.readonly`）。
  - クライアントIDを発行し、ローカルの `.env.local` に `VITE_GOOGLE_CLIENT_ID` として保存。
- [ ] **2-2. OAuth 2.0 ログインの実装**
  - `@react-oauth/google` などのライブラリ、または生のFetchでフロントエンドのみの認証フローを実装。
  - ログイン成功時に `accessToken` を取得し、ブラウザのメモリ（State）に一時保存する。
- [ ] **2-3. カレンダー一覧の取得 (`calendarList`)**
  - ログイン完了後、ユーザーが持つ全カレンダーを取得。
  - 左側サイドバーの shadcn/ui `Checkbox` リストにカレンダー名を表示し、複数選択可能にする。

---

## 📅 フェーズ 3: データマージ & マンスリー描画

選択された複数のカレンダーから予定を引き抜き、1つのマンスリーカレンダーに美しく統合します。

- [ ] **3-1. 複数カレンダーの予定を並列取得**
  - 選択されたカレンダーIDの配列をもとに、`Promise.all` で各カレンダーのイベント（表示対象月）を同時にFetch。
  - 取得したイベントを1つの配列にマージ。この時、親カレンダーの色（`backgroundColor`）の情報を予定データに付与する。
- [ ] **3-2. フロントエンド側での予定ソートロジック**
  - 1つの日付の中に混在する複数カレンダーの予定を、「① 終日予定が上」「② 開始時間が早い順」に並び替えるロジックを実装。
- [ ] **3-3. CSS Grid によるマンスリーグリッドの描画**
  - 縦6行×横7列のグリッドを作成し、日付とソート済みの予定を描画。
  - 長い予定タイトルが枠を突き抜けないよう、Tailwindの `truncate`（`text-overflow: ellipsis`）で文字溢れ対策。

---

## 🖨️ フェーズ 4: A4印刷最適化 (CSS @media print)

このアプリの最重要価値である「綺麗にA4に収まる印刷ビュー」を作り込みます。

- [ ] **4-1. 印刷用アスペクト比の固定**
  - カレンダーの表示エリアに対し、A4横の比率（`aspect-[1.414/1]`）を適用。
  - 印刷時（`print:` プレフィックス）に幅と高さを A4サイズ（`w-[297mm] h-[210mm]`）にカチッと固定。
- [ ] **4-2. `@media print` スタイルの調整**
  - 印刷実行時に、左側の操作サイドバーや印刷ボタンなど「カレンダーの箱以外」を `visibility: hidden`（または `display: none`）で完全に消し去るCSSを記述。
  - ブラウザの印刷設定で背景色が消えないよう、`print-color-adjust: exact` を指定。
- [ ] **4-3. 印刷プレビューのテスト調整**
  - 実際に `window.print()` を発火させるボタンを配置。
  - Chrome等の開発者ツール（Rendering -> Emulate CSS media type: print）を使い、枠線や文字サイズ（`text-[10px]`等）がA4 1ページに美しく収まるか微調整。

---

## 🚀 フェーズ 5: Netlify デプロイ & 最終確認

Netlifyの強みであるAI分析を活用しながら、世界に公開します。

- [ ] **5-1. Netlify 上での環境変数設定**
  - Netlifyの管理画面（Site configuration -> Environment variables）に移動。
  - `VITE_GOOGLE_CLIENT_ID` を登録。
  - 予期せぬビルドエラーを防ぐため、`NODE_VERSION`（例: `22`）も環境変数に指定。
- [ ] **5-2. リポジトリ連携 & 初回デプロイ**
  - GitHubにリポジトリをプッシュし、Netlifyと連携。
  - 初回ビルドを実行。もしエラーが出た場合は、**NetlifyのAIビルド分析フィードバック**を確認して修正。
- [ ] **5-3. Google Cloud Console への本番URL登録**
  - Netlifyから発行された本番URL（`https://xxx.netlify.app`）を、Google CloudのOAuthクライアント設定の「承認されたJavaScript生成元」および「承認されたリダイレクトURI」に追加。
- [ ] **5-4. 実機（紙・PDF）テスト**
  - 本番環境にアクセスし、実際に複数カレンダーを選択してA4用紙（またはPDF保存）に一発で綺麗に印刷できるか最終確認。

## パターン1：【README / LP用】最も標準的で洗練されたコンセプト

> ### JFPYG cal (Just for printing your google calendar)
>
> **「ただ、Googleカレンダーを美しく印刷するためだけに。」**
> JFPYG calは、GoogleカレンダーのデータをA4サイズでの印刷・PDF出力に100%最適化して描画する、ステートレスなマンスリーカレンダー型Webビューアです。
> 「仕事、プライベート、家族の予定など、複数のカレンダーを重ね合わせて紙で持ち歩きたい」「標準の印刷機能だとレイアウトや色味が崩れてしまう」という課題を、独自のA4比率グリッドデザインと、フロントエンド側での高度な予定ソートロジックによって解決します。
> データベースを一切持たない（ステートレスな）設計のため、あなたのカレンダー情報が外部のサーバーに保存されることはありません。ログインしたその瞬間だけ、安全かつ最速で「最高の印刷用カレンダー」を出力します。

## Pattern 2: 【開発者/ポートフォリオ用】技術的な美しさを押し出すコンセプト

> ### JFPYG cal — データベースを持たない、高セキュリティ・高機能カレンダープリンター
>
> **【プロダクトの核心】**
> 本プロダクトは、データの永続化（DB）をあえて行わないことで、究極のデータプライバシーと軽量な動作を実現した、Googleカレンダー専用のA4マンスリー印刷最適化ツールです。
> **【技術的アプローチと解決する課題】**
>
> 1. **セキュリティファースト（No DB）:** OAuth 2.0 クライアントサイドフローを採用し、アクセストークンやカレンダーデータをブラウザのメモリ内のみで処理。サーバーサイドに一切の個人情報を残さないセキュアなアーキテクチャ。
> 2. **複数データソースの統合（Multi-Calendar Merge）:** ユーザーが選択した複数のカレンダーAPIから `Promise.all` を用いてイベントを並列取得し、「終日予定」「時間順」のソートアルゴリズムを適用して1つの美しいマンスリーグリッドにマージ。
> 3. **ピクセルパーフェクトな印刷制御（CSS @media print）:** ブラウザの印刷ダイアログの癖を徹底的に排除し、A4横（Landscape）の比率に完全にフィッティング。文字溢れの自動省略や、背景色の印刷色維持（color-adjust）をCSS Gridで制御。
